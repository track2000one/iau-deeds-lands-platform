# الوضع التجريبي | Demo Mode

## 🚀 ما هو الوضع التجريبي؟

الوضع التجريبي يتيح لك **تجربة المنصة بدون الحاجة لإعداد Firebase**. يمكنك:
- تسجيل الدخول بحسابات تجريبية
- تصفح جميع واجهات المنصة
- تجربة الميزات الأساسية
- البيانات تُحفظ محلياً في المتصفح (localStorage)

## 👥 الحسابات التجريبية

### 1️⃣ حساب المسؤول
- **البريد الإلكتروني**: `admin@demo.com`
- **كلمة المرور**: `admin123`
- **الصلاحيات**: جميع الصلاحيات (إضافة، تعديل، حذف، طباعة)

### 2️⃣ حساب الموظف
- **البريد الإلكتروني**: `employee@demo.com`
- **كلمة المرور**: `employee123`
- **الصلاحيات**: محدودة (عرض فقط افتراضياً)

---

## ✅ مميزات الوضع التجريبي

- ✨ لا يحتاج إلى إعداد Firebase
- 🚀 جاهز للاستخدام فوراً
- 💾 البيانات تُحفظ محلياً
- 🔐 نظام مصادقة بسيط
- 📱 تجربة كاملة للمنصة

---

## ⚠️ قيود الوضع التجريبي

- 🔒 **بيانات محلية فقط**: البيانات تُحفظ في المتصفح وليس في قاعدة بيانات
- 👤 **حسابات محدودة**: حسابان تجريبيان فقط (مسؤول وموظف)
- 🗑️ **البيانات تُحذف**: عند مسح ذاكرة التخزين المؤقت (cache)
- 👥 **مستخدم واحد**: لا يمكن تعدد المستخدمين
- 📎 **لا يمكن رفع ملفات**: نظام المرفقات غير متاح

---

## 🔄 التبديل من الوضع التجريبي إلى Firebase

عندما تكون مستعداً للاستخدام الفعلي:

1. **اتبع دليل الإعداد**: راجع [FIREBASE_SETUP.md](./FIREBASE_SETUP.md) أو [QUICK_START.md](./QUICK_START.md)
2. **أنشئ مشروع Firebase**
3. **أضف بيانات Firebase في `.env`**
4. **أعد تشغيل المشروع**

المنصة ستنتقل تلقائياً من الوضع التجريبي إلى Firebase!

## 🎯 كيف يعمل الوضع التجريبي؟

### المصادقة
```typescript
// في lib/demoAuth.ts
- تسجيل الدخول بالبريد وكلمة المرور
- حفظ الجلسة في localStorage
- تحميل المستخدم عند فتح الصفحة
```

### البيانات
```typescript
// البيانات تُحفظ في localStorage
- الصكوك
- الأراضي
- المباني
- جميع السجلات الأخرى
```

### التحقق التلقائي
```typescript
// في config/firebase.ts
if (!isFirebaseConfigured()) {
  // استخدام الوضع التجريبي
} else {
  // استخدام Firebase
}
```

## 📝 ملاحظات مهمة

### ✅ للتطوير والتجربة
الوضع التجريبي مثالي لـ:
- تجربة المنصة قبل الإعداد الكامل
- عرض المنصة للآخرين
- التطوير المحلي السريع

### ⚠️ ليس للإنتاج
**لا تستخدم** الوضع التجريبي في بيئة الإنتاج:
- البيانات غير محمية
- لا يوجد نسخ احتياطي
- البيانات يمكن فقدها بسهولة

### 🔐 للاستخدام الفعلي
للاستخدام في بيئة حقيقية، يجب إعداد Firebase:
- قاعدة بيانات آمنة
- نظام مصادقة قوي
- رفع ملفات محمي
- نسخ احتياطية تلقائية
